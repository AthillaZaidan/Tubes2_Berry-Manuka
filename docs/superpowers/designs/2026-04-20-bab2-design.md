# Bab 2 Design — Landasan Teori

## Overview
Bab 2 akan berisi 10 section dengan struktur yang mengikuti gaya penulisan referensi Tucil2_13524061_13524068.

## Struktur Section

### 2.1 Document Object Model (DOM)
- Definisi DOM sebagai representasi pohon dokumen HTML
- Struktur hierarkis: root, parent-child, sibling relationships
- Node types: element, text, attribute nodes
- Manipulasi DOM dalam konteks web browser

### 2.2 HTML dan HTML Parsing
- Definisi HTML sebagai markup language
- Struktur HTML: tags, attributes, elements
- Proses parsing: tokenizer → DOM tree construction
- Void elements dan self-closing tags
- Parser implementations (golang.org/x/net/html)

### 2.3 CSS Selector
- Konsep selector untuk memilih elemen DOM
- Tipe selector: tag, class, ID, universal
- Combinators: child (>), descendant (space), adjacent sibling (+), general sibling (~)
- Cara kerja matching selector pada pohon DOM

### 2.4 Breadth-First Search (BFS)
- Definisi algoritma traversal level-by-level
- Implementasi dengan queue (FIFO)
- Pseudocode BFS untuk pohon
- Karakteristik: time complexity O(N), space complexity O(W)
- Kapan BFS optimal untuk DOM traversal

### 2.5 Depth-First Search (DFS)
- Definisi algoritma traversal depth-first
- Implementasi dengan stack (LIFO) atau rekursi
- Pseudocode DFS untuk pohon (pre-order)
- Karakteristik: time complexity O(N), space complexity O(H)
- Kapan DFS optimal untuk DOM traversal

### 2.6 Perbandingan BFS dan DFS
- Tabel perbandingan komprehensif
- Perbedaan urutan kunjungan node
- Perbandingan penggunaan memori (lebar vs kedalaman pohon)
- Skenario penggunaan yang tepat untuk masing-masing

### 2.7 Web Scraping
- Definisi web scraping
- Teknik pengambilan data dari URL
- HTTP client dan request handling
- Timeout dan error handling
- Validasi content-type dan security considerations

### 2.8 Lowest Common Ancestor dan Binary Lifting
- Definisi LCA dalam konteks pohon
- Pendekatan naive vs Binary Lifting
- Preprocessing O(N log N) dengan tabel ancestor 2^k
- Query O(log N) dengan teknik binary lifting
- Implementasi path reconstruction

### 2.9 Konkurensi dan Goroutine
- Konsep concurrency vs parallelism
- Goroutine di Go: lightweight threads
- WaitGroup untuk sinkronisasi
- Mutex untuk thread safety
- Channel untuk komunikasi antar goroutine
- Implementasi parallel BFS dan DFS

### 2.10 Containerization dan Docker
- Konsep containerization
- Docker sebagai platform container
- Dockerfile: multi-stage builds
- Docker Compose untuk multi-container apps
- Deployment dengan container orchestration

## Writing Style Guidelines

1. **Bahasa:** Indonesia formal akademik
2. **Format:** Latex dengan package yang sama (algorithm, algpseudocode, dll)
3. **Citations:** Gunakan referensi dari dafpus.bib (cormen2009introduction, mdn_dom, dll)
4. **Structure:** Paragraph pembuka → subsections dengan \subsection → list/deskripsi
5. **Math notation:** Gunakan equation/align untuk complexity analysis
6. **Code:** Pseudocode dalam environment algorithm
